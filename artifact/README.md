# Artifact Reproducibility Requirements

To ensure that the artifacts generated by this project are reproducible, the following requirements must be met:

1. **Environment Setup**
   - Use a consistent programming environment with the specified version of Python and dependencies.
   - It is recommended to use virtual environments (e.g., `venv` or `conda`) to isolate the project dependencies.

2. **Data Management**
   - All datasets used for training and testing must be accessible and preferably version-controlled.
   - Document the data sources and how to obtain them in this README.

3. **Parameterization**
   - The models should be parameterized in a way that allows for easy modifications without changing the core codebase.
   - Provide details about the parameters used in the experiments.

4. **Dependency Management**
   - Create a `requirements.txt` file or equivalent that lists all the necessary packages to reproduce the environment.

5. **Version Control**
   - Use Git for version control of both code and data.

# How to Run the Demo

To run the demo, follow the steps below:

1. **Clone the Repository**
   ```bash
   git clone https://github.com/anjoravimac-ctrl/luminaq-paper.git
   cd luminaq-paper
   ```

2. **Set Up the Environment**
   - Make sure you have Python installed, then create and activate the virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use `venv\Scripts\activate`
   ```
   - Install the required dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. **Run the Demo**
   - Execute the demo script:
   ```bash
   python demo.py
   ```
   - Follow any printed instructions to interact with the demo.

Make sure to refer to the contributing guidelines for detailed steps if you want to contribute to this project.
