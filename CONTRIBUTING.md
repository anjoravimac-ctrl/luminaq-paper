# Contribution Guidelines

Thank you for considering contributing to the LuminaQ project! Here are some guidelines to help you get started:

## How to Contribute
1. **Fork the repository**: Click the "Fork" button at the top right of the repository page.
2. **Clone your fork**: Use `git clone https://github.com/YOUR_USERNAME/luminaq-paper.git` to clone your fork to your local machine.
3. **Create a new branch**: Use `git checkout -b your-feature-branch` to create a new feature branch.
4. **Make your changes**: Implement your changes or fixes.
5. **Commit your changes**: Use `git commit -m "Brief description of your changes"` to commit your changes.
6. **Push to your fork**: Use `git push origin your-feature-branch` to push your changes to your fork on GitHub.
7. **Open a Pull Request**: Go to the original repository and click on "New Pull Request". Select your feature branch and submit the pull request.

## Code Style
- Follow the existing code style in the repository.
- Use meaningful variable and function names.

## Issues and Feedback
- If you encounter issues or have suggestions, feel free to create an issue in the repository.

## License

Thank you for contributing!
